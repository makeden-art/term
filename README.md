# DMTerm

Небольшой клиент под Linux: **SSH вкладки + локальный shell + SFTP**, рассчитан на Astra/Debian‑окружение. В качестве SSH используется **системный `ssh`**.

## Что умеет

### Терминал

- **ПКМ в терминале как в Windows Terminal**:
  - есть выделение → копировать в буфер;
  - нет выделения → вставить из буфера.
- **Bracketed paste** для больших вставок (удобнее в `nano`/`vim`, меньше «съезжания» ввода).
- **Панель «Команды»** (справа): сохранённые многострочные команды, вставка в активный PTY; после запуска команды фокус возвращается в терминал.
  - **Группы команд** (сворачиваемые), добавление / правка / удаление.
  - Отдельные **«Выгрузить…» / «Загрузить…»** для обмена списком команд (JSON `dmterm-commands`), без связи с сессиями SSH.

### Вкладки и раскладка

- Вкладки **SSH** и **Local**; переключение между сессиями.
- **Перетаскивание вкладок** для смены порядка.
- **Слияние во вкладку (mux)** с общим вводом (**broadcast**): перетащите вкладку на другую с **Ctrl** (**Cmd** на macOS) — несколько терминалов стеком в одной вкладке, клавиатура уходит во все панели.
- **Split**: перетаскивание **без** Ctrl — два терминала **слева и справа** в одной вкладке (общий ввод только в активную панель).
- У **split** можно **закрыть объединённую вкладку** так, что панели разъедутся обратно на отдельные вкладки (PTY остаются живыми).
- **Закрытие вкладки средней кнопкой мыши** (как в браузере).
- **Восстановление закрытой вкладки**: `Ctrl+Shift+T`.
- **Восстановление набора вкладок** после перезапуска приложения.
- Верхнее меню (**File / Edit / View / Window / Help**): выход, масштаб, полноэкранный режим, DevTools, свернуть/закрыть окно и т.д.

### Сессии SSH и список слева

- **Группы хостов** (сворачиваются по стрелке).
- **Подгруппы** внутри группы (сворачивание, поле в форме сессии).
- **Jump host** (`-J` / bastion) и **локальный проброс портов** (`-L`, поле Local forward).
- Аутентификация: **пароль** или **ключ** (путь к файлу; можно выбрать файл из `~/.ssh`).
- Отдельный **ключ для jump** (если нужен).
- **X11 forwarding** и вариант **trusted**.
- **ПКМ по сессии**: дублировать запись (**Копировать** без копирования секретов в конфиг), открыть **настройки**, открыть **SFTP**, **удалить**.
- **ПКМ по группе/подгруппе**: **Открыть** все SSH сессии в области **по отдельным вкладкам**; войти в **мульти‑режим** с предвыбором.
- **Мульти‑режим**: чекбоксы по группам/хостам; **Enter** открывает **одну mux‑вкладку** со всеми выбранными сессиями и общим вводом; **Escape** — выход из режима.
- Секция **«Новая сессия»** (можно свернуть): создание и правка профиля, группа, подгруппа, jump, tmux и пр.

### SFTP

- Панель **SFTP** (листинг, скачивание, загрузка) для активной сессии.
- Доступ к секретам через тот же **vault**, что и SSH.

### tmux

- Для **SSH**: при включении tmux в сессии — `tmux new -A -s <имя>`; модальное окно для выбора/создания имени сессии.
- Для **Local**: опция **Local tmux** с сессией **`dmterm_local`**.

### Хранилище (vault) и блокировка

- **Мастер‑пароль**: секреты **шифруются** (отдельный зашифрованный блок в конфиге), пароли и ключи не хранятся открытым текстом в файле сессий.
- **Автоподстановка пароля SSH** по приглашению вида `user@host's password:` (в том числе цепочка с **-J**).
- **Блокировка по простоя** (таймер «Лок экрана», минуты): интерфейс скрывается за вводом мастер‑пароля; уже открытые SSH/tmux продолжают работать.
- **Смена мастер‑ключа**: проверяется старый пароль; содержимое vault **перешифровывается** новым ключом и новой солью.ф

### Экспорт / импорт данных

- **«Выгрузить…» / «Загрузить…»** в боковой карточке:
  - режим переноса: отметить группы/хосты и выгрузить **только профили сессий (без паролей и приватных ключей)**;
  - импорт **дополняет** список сессий.
- В панели **«Команды»** — отдельный экспорт/импорт файла **`dmterm-commands`** (названия, текст команды, группа).

### Интерфейс списка

- **Компактный режим** сайдбара: узкая колонка с иконками, группы и подгруппы свёрнуты по необходимости; **кнопка «Команды»** в шапке.
- Блок настроек активной сессии, SFTP, команды — доступны из кнопок внизу карточки (SFTP · Команды · Настройки в одном ряду).

## Vault и пароли

- Пароли и ключи **не** зашиваются в AppImage и **не** лежат в репозитории.
- Данные приложения: каталог **`DMTerm`** в конфиге пользователя (на Linux обычно **`~/.config/DMTerm/`**), файл **`dmterm.config.json`** — сессии, группы и **зашифрованный vault**.
- После ввода **мастер‑пароля** приложение может подставлять секреты в **SSH**, **SFTP** и **авто‑ответ** на password‑prompt.

## Перенос на другую машину

- Передать **`AppImage`** и при необходимости скопировать **`~/.config/DMTerm/`** (там и сессии, и vault).
- Либо только AppImage и перенести **список хостов** через **«Выгрузить…» / «Загрузить…»** — в файл **не попадают** пароли и ключи; их нужно заново сохранить в vault на новой машине или перенести вместе с каталогом конфигурации.
