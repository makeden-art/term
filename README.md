# DMTerm

Небольшой “mobaXterm‑like” клиент под Linux: **SSH вкладки + SFTP панель**, рассчитан на Astra/Debian‑окружение.

### Что умеет

- **SSH вкладки** (используется системный `ssh`)
- **SFTP панель** справа (листинг/скачивание/загрузка)
- **ПКМ в терминале как в Windows Terminal**:
  - если есть выделение → копировать
  - если нет выделения → вставить
- **Вставка больших кусков текста** в редакторы (bracketed paste; меньше “съезжает” в `nano`)
- **Группы хостов** (сворачиваются стрелкой)
- **Jump host (-J)** и **Local forward (-L)** (через поля сессии)
- **Перетаскивание вкладок** и **split‑режим**:
  - перетаскивайте вкладку, чтобы переставить
  - перетащите вкладку на другую (в центр) → два терминала рядом в одном окне
- **Закрытие вкладки колесом** (middle click), как в браузере
- **Восстановление закрытой вкладки**: `Ctrl+Shift+T` (как в браузере)
- **Восстановление открытых вкладок** после перезапуска приложения (вкладки поднимаются заново)
- **Свертывание левой панели** (компактный режим; группы сворачиваются; элементы читаемые)
- **Хранилище секретов (vault)**: мастер‑пароль → пароли/ключи шифруются и не лежат “как текст”
- **Автоподстановка SSH‑пароля** по prompt `user@host's password:` (включая `-J`)
- **Экспорт/импорт** групп и хостов (без паролей)
- **tmux**:
  - для SSH: `tmux new -A -s <name>` (если включено в настройках сессии)
  - для Local: опционально `tmux new -A -s dmterm_local`

## Запуск в приложения

```
chmod +x release/*.AppImage
./release/*.AppImage
```
## Запуск в dev

Требуется **Node.js 22+**.

```bash
npm install
npm run dev
```

Если Electron не запускается и пишет `Missing X server or $DISPLAY`, значит вы в headless‑сессии (без GUI).
Запускайте из графической среды Astra или через `xvfb-run`.

## Сборка portable

### Linux / Astra (AppImage)

Сборку AppImage лучше делать **на Linux** (на самой Astra или в VM/WSL2 Debian/Ubuntu).

```bash
npm install
npm run build
npm run pack:linux
```

Результат будет в `release/`.

Запуск:

```bash
chmod +x release/*.AppImage
./release/*.AppImage
```

### Windows (portable exe)

```bash
npm install
npm run build
npm run pack:win
```

## Vault / пароли

- Пароли/ключи **не зашиваются в AppImage** и не хранятся в репозитории.
- Данные приложения лежат в профиле пользователя (обычно `~/.config/DMTerm/`), там же `dmterm.config.json` с зашифрованным vault.
- Вы вводите **мастер‑пароль** один раз (разблокировка vault), дальше приложение может:
  - подставлять пароль/ключ для SFTP
  - подставлять пароль в `ssh` (когда видит prompt `... password:`)

## Перенос на другую Astra

Самый простой путь — передать `.AppImage` и запустить.  
Если нужно перенести список хостов:

- в приложении есть **“Выгрузить…” / “Загрузить…”** (экспорт/импорт групп и хостов)
- пароли в экспорт **не попадают** (их переносите отдельно через vault/мастер‑пароль, если нужно)
